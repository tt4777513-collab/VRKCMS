# Система Аутентификации CMS Pro

## Описание
Система хранит учетные записи пользователей в браузерной базе данных (`localStorage`). При входе или регистрации проверяются учетные данные.

## Функции

### Регистрация
1. Пользователь нажимает "НАЧАТЬ БЕСПЛАТНО"
2. Вводит данные:
   - **Имя**: любое значение
   - **Email**: минимум 10 символов, должен быть валидный email формат (example@mail.com)
   - **Пароль**: минимум 10 символов
   - **Подтверждение пароля**: должно совпадать с паролем

3. При успешной регистрации:
   - Аккаунт сохраняется в `localStorage` (ключ: `cmsPro_accounts`)
   - Создается сессия пользователя (ключ: `userSession`)
   - Пользователь перенаправляется на админ-панель

### Вход
1. Пользователь нажимает кнопку "Вход" в модальном окне
2. Вводит данные:
   - **Email**: зарегистрированный email
   - **Пароль**: правильный пароль
   - **Запомнить меня**: опционально

3. При входе:
   - Система проверяет, существует ли аккаунт с таким email
   - Проверяет, совпадает ли пароль
   - Если данные неправильные, показывается сообщение: "Неправильный логин или пароль"
   - Если данные правильные, создается сессия и пользователь перенаправляется на админ-панель

### Выход
1. На админ-панели нажать "Выход" в боковом меню
2. При подтверждении выхода:
   - Сессия пользователя удаляется
   - Пользователь перенаправляется на главную страницу

## Demo Учетная Запись

При первом запуске система автоматически создает demo аккаунт:
- **Email**: `demo@example.com`
- **Пароль**: `DemoPassword123`

Используйте эту учетную запись для проверки функциональности.

## Структура Данных

### Аккаунты (cmsPro_accounts)
```json
[
  {
    "id": 1,
    "email": "user@example.com",
    "password": "Password123",
    "name": "User Name",
    "createdAt": "2026-06-17T10:00:00.000Z"
  }
]
```

### Сессия (userSession)
```json
{
  "id": 1,
  "email": "user@example.com",
  "name": "User Name",
  "loginTime": "2026-06-17T10:00:00.000Z",
  "rememberMe": true
}
```

## Функции JavaScript

### main.js

#### `initAccountsDatabase()`
Инициализирует базу данных аккаунтов (создает demo аккаунт если база пуста)

#### `getAllAccounts()`
Возвращает все аккаунты из базы данных

#### `saveAccount(email, password, name)`
Сохраняет новый аккаунт
- Возвращает: `{ success: boolean, message: string }`

#### `checkAccount(email, password)`
Проверяет наличие аккаунта и корректность пароля
- Возвращает: `{ success: boolean, message: string, account?: object }`

#### `validateAuthFields(email, password)`
Валидирует поля email и пароль
- Возвращает: `null` если валидно, или строку ошибки

### admin.js

#### `checkUserAuthentication()`
Проверяет, залогинен ли пользователь, перенаправляет если нет

#### `getAllAccounts()`
Возвращает все аккаунты из базы данных

#### `getAccountByEmail(email)`
Получает аккаунт по email адресу

## Правила Валидации

### Email
- ✓ Минимум 10 символов
- ✓ Должен быть в формате: example@mail.com
- ✓ Не может быть дублирован (уникальность)

### Пароль
- ✓ Минимум 10 символов
- ✓ Должен совпадать при регистрации

## Безопасность

⚠️ **Важно**: Текущая система хранит пароли в чистом виде. Это предназначено только для разработки и тестирования.

**Для production:**
1. Используйте HTTPS
2. Хешируйте пароли на сервере (используйте bcrypt или аналогичное)
3. Никогда не отправляйте пароли в открытом виде
4. Используйте JWT токены для сессий
5. Добавьте защиту от CSRF атак
6. Используйте двухфакторную аутентификацию

## Тестирование

1. **Регистрация**: Создайте новый аккаунт с валидными данными
2. **Вход с демо**: Используйте demo@example.com / DemoPassword123
3. **Ошибка входа**: Попробуйте войти с неправильными данными
4. **Выход**: Проверьте выход из админ-панели
